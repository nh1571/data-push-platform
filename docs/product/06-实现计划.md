# 企业数据推送中台 Implementation Plan

> **For agentic workers:** 推荐使用 `subagent-driven-development` 或 `executing-plans` 按 Task 推进。步骤用 `- [ ]` 跟踪。  
> **设计依据：** [[01-需求调研]] · [[02-领域模型]] · [[03-架构设计]] · [[04-UI信息架构]] · [[05-UI视觉设计]]

**Goal:** 交付一期完整系统（成功标准 A–F）：Doris/MySQL 取数 → 四形态渲染 → 钉钉通道；自建调度 + DS HTTP 触发；执行日志与重跑；管理台可用。

**Architecture:** 单一 Execution 流水线；DataSource / Renderer / Channel 三套插件 SPI；API + Worker + Scheduler 分进程；元数据 PostgreSQL；触发入口统一 `Run`。

**Tech Stack:**

| 层 | 选型 |
|----|------|
| 语言 | Python 3.12+ |
| API | FastAPI + Pydantic v2 + Uvicorn |
| ORM/迁移 | SQLAlchemy 2.x + Alembic |
| 元数据库 | PostgreSQL 16 |
| 队列 | Celery + Redis |
| 业务取数 | PyMySQL / mysqlclient；Doris 走 MySQL 协议 |
| 前端 | React 18 + Vite + TypeScript + Ant Design 5（对齐视觉通用蓝） |
| 部署 | Docker Compose（api / worker / scheduler / postgres / redis） |
| 测试 | pytest + httpx；前端 vitest（按需） |

**建议仓库名：** `data-push-platform`（可改）

**流水线顺序（不可改乱）：**

1. 创建 JobRun `pending` → 入队  
2. Worker：`running` + 写入 config_snapshot  
3. 取数 → 若 `skip_if_empty` 且无行 → `succeeded`，Delivery `skipped` 或不建  
4. 渲染 MessagePart[]  
5. 各 Channel send → Delivery  
6. 汇总 status：`succeeded` / `failed` / `partial`

## A–F 验收清单（发布前）

| ID | 检查项 | 验证方式 |
