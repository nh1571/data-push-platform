# 协作约定与架构路径

## 用户授权（2026-07-16）

> 按照项目整体架构师的设计路径进行开发；中途任何建议 Agent **可接纳或否定**，并更新计划与文档；**整体前行路径由 Agent 把控**。

## 路径主线（有序）

1. **内容生产核心：组件画板设计器** — 数据集 + 组件 + 固定宽画板 + 编译成图/文案（见 [[17-组件画布-内容生产核心设计]]）  
2. **成图 / 富文本双预览闭环**（与 compile 同一路径）  
3. **发图可用通道** — OpenAPI 群/单发（搭配层）  
4. **数据源扩展** — SQL Server 等  
5. **运维闭环** — 已读/撤回/退订等  

支线（通道 UI、调度花活）不得长期压过 1–2。  
**否定**：一上来做 Figma 式无限自由画布；做成完整帆软（填报/大屏/权限）全家桶。

## 建议处理规则

| 用户建议 | Agent 动作 |
|----------|------------|
| 与主线一致 | 纳入当前迭代，更新 HANDOFF §6 |
| 有价值但非当前 | 记入延后清单，主线不动 |
| 与原则冲突（如回到脚本堆任务） | **否定**并说明，写进笔记 |
