# 组件组装交互模型

## 用户纠正（2026-07-17）

最终产物 = **推送内容**，由多个 **组件** 组成。

```text
数据（数据集/SQL/字段）
    ↓ 绑定
组件实例（饼图/柱图/表/文案… + 样式 + 绑定）
    ↓ 组装
推送内容（有序组件树 → 编译成图/文案）
    ↓
通道投递
```

**不是**：在一张模糊画板上乱点，把「列名」当成展示内容。

## 错误交互（已否定）

- 绑定后 KPI/图上显示的是 **字段标题**，不是 **数据值**
- 数据、绑定、成片搅在同一屏，心智混乱

## 正确四步

| 步 | 名称 | 用户在做什么 |
|----|------|----------------|
| 1 | **数据** | 准备数据集：源、SQL、取数、看字段 |
| 2 | **组件** | 添加组件类型 → 设样式 → **字段拖到槽位** → 看该组件的**实值预览** |
| 3 | **组装** | 调整组件顺序/容器，决定推送里谁上谁下 |
| 4 | **预览** | 整页编译成图/文案，保存任务、试推 |

## 组件生命周期

1. **选型**：柱状图 / 饼图 / 表 / KPI / 文案 / 告警…  
2. **样式**：主题色、表风格、图表类型…  
3. **绑定**：数据集 + 字段映射（帆软式拖字段）  
4. **成件**：绑定成功后显示 **真实数据**，不是列名  
5. **入编**：进入组装顺序
