# 模板与运行时动态渲染

> 用户确认（2026-07-17）：整体思路正确；工作台只是做模版，每次推送按预置参数动态渲染图片再推送。

## 一句话

**编辑 = 定模板；推送 = 用当日/当次参数重新取数、重新成图、再投递。**

## 编辑态（内容工作台）

保存到 `PushJob` 的是 **artboard 模板**，不是某次运行的 PNG：

| 存什么 | 说明 |
|--------|------|
| 数据集 SQL | 可含 `{{yesterday}}` 等占位 |
| 参数定义 | auto / static / runtime |
| 组件树 | 类型、字段绑定、画布 x/y/w/h/风格 |
| 推送壳 | title、图前/图后富文本（可含 `{{字段}}`） |
| 通道 / 调度 | 任务级配置 |

**不存**「永远有效」的业务数值截图作为唯一成品。

## 运行态（JobRun / 试推 / 调度）

每次执行：

```text
参数解析（sql_params）
  → 数据源插件 execute（实时 SQL）
  → artboard_to_message（组件 HTML → Playwright 成图 + 文案壳）
  → 通道 send
```

因此：

- 昨天的「昨日门诊量」与今天的「昨日门诊量」是**两次不同渲染**；
- 工作台预览只是用**当前样例数据**做「像不像」验收；
- 改模板版式后，下一次推送自动用新版式 + 新数据。

## 与交互五步的关系

1. **数据** — 写模板 SQL 与参数（样例取数方便绑字段）  
2. **做组件** — 绑定列名/展示形态  
3. **组装画布** — 模板版式（推送图骨架）  
4. **组装推送** — 图外文案模板  
5. **预览推送** — 样例预演；试推走完整运行路径  

## 代码锚点

- `backend/app/modules/execution/pipeline.py` — JobRun 动态取数 + 渲染  
- `backend/app/modules/studio/service.py` — compile / test-push  
- `backend/app/modules/studio/sql_params.py` — 参数每次重算  
- `backend/app/modules/studio/compile.py` — 模板 → Message
