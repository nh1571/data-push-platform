# 交互理念重设计

## 用户反馈

「推送编辑」打开却是「请先去任务列表建任务」——交互理念有问题，功能分配要重做。

## 旧设计问题

| 设计 | 问题 |
|------|------|
| 内容工作台必须先有任务实体 | 日常 80% 是「改稿」，却被任务管理拦住 |
| 菜单「推送编辑」与「推送任务」语义打架 | 不知道从哪进 |
| 先管理、后内容 | 与「内容生产主战场」原则相反 |

## 新理念（内容优先）

```text
内容工作台（/editor）  = 日常主入口
  可空白开工：取数 → 设计 → 预览 → 试推
  保存 → 创建/更新「任务」

任务管理（/push-jobs） = 运营/调度
  列表、启用、最近运行、执行、进编辑
  新建任务（可选快捷入口）→ 也可只打开工作台
```

## 实现要点

- 去掉「无 jobId 整页拦截」
- `/editor` 空白可编辑；「保存为任务」调用 save-job 创建
- 侧栏文案：内容工作台 / 任务管理
- 任务列表说明职责边界

## 变更

- 2026-07-16：按用户截图反馈重设计并改前端
