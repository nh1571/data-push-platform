# UI 视觉设计（一期）

> 前置：[[04-UI信息架构]]（结构已确认）。  
> 本文定 **观感、布局骨架、关键屏效果约定**，不是切图稿；实现时可用 Ant Design / shadcn 等落地。  
> 产品类型：**企业内部数据工具**，优先清晰、密度适中、状态可读，拒绝营销风渐变花活。

---

## 1. 产品气质

| 维度 | 选择 |
|------|------|
| 调性 | 专业、冷静、可信的数据中台后台 |
| 密度 | **中等**：表格与表单信息量大，但不拥挤到失控 |
| 色彩角色 | 中性灰主导 + 一条品牌主色 + 语义色（成功/警告/危险/信息） |
| 动效 | 极少；加载与折叠即可，不做炫技过渡 |
| 插画/装饰 | 基本不用；空态用简短文案 + 主按钮 |

**一句话**：像「好用的运维/数仓控制台」，不像 Landing Page。

---

## 2. 布局骨架（所有页统一）

```text
┌──────────────────────────────────────────────────────────┐
│ 顶栏 56px：产品名 | 当前环境(可选) | 用户/退出              │
├────────────┬─────────────────────────────────────────────┤
│            │  内容顶：页面标题 + 主操作按钮（右上）         │
│  侧栏      │─────────────────────────────────────────────│
│  220px     │                                             │
│  可折叠至  │           主内容区（白底卡片/表格）            │
│  64px 图标 │                                             │
│            │                                             │
└────────────┴─────────────────────────────────────────────┘
```

| 区域 | 规格建议 |
|------|----------|
| 侧栏宽度 | 展开 220px / 收起 64px |
| 顶栏高度 | 56px |
| 内容左右边距 | 24px |
| 内容最大宽度 | 不强制居中限宽（表格要吃满）；表单单栏最大约 720–960px |
| 卡片圆角 | 8px |
| 基础字号 | 14px 正文；12px 辅助；16–20px 页标题 |

**侧栏**

- 背景：略深于内容区（浅灰或极浅蓝灰），与内容区分层
- 当前项：左侧 3px 主色条 + 轻微底色
- 图标 + 文案；收起时仅图标 + Tooltip

**顶栏**

- 白底 + 底部分割线
- 左：`数据推送`（或最终产品名）
- 右：用户名下拉（退出）

---

## 3. 色彩（语义优先）

不绑定具体品牌色代码时可先定角色；实现时再填 token。

| Token | 用途 | 建议方向 |
|-------|------|----------|
| `color-bg-page` | 页面底 | 浅灰 `#F5F6F8` 一类 |
| `color-bg-surface` | 卡片/表格容器 | 白 |
| `color-border` | 分割、输入框 | 浅灰边 |
| `color-text` | 主文案 | 近黑 |
| `color-text-secondary` | 说明、时间 | 中灰 |
| `color-primary` | 主按钮、链接、侧栏选中 | **蓝系**（工具台常见，可换成公司色） |
| `color-success` | succeeded | 绿 |
| `color-warning` | partial / 运行注意 | 橙 |
| `color-error` | failed、删除 | 红 |
| `color-info` | running / pending | 蓝/青 |

**状态徽章（执行状态）**

| 状态 | 样式 |
|------|------|
| succeeded | 绿底浅 + 绿字 或 描边徽章 |
| failed | 红 |
| partial | 橙 |
| running | 蓝 + 可选小 spinner |
| pending | 灰 |

同一套徽章在列表与详情复用。

## 4. 字体与层级

| 层级 | 用法 |
|------|------|
| 页标题 | 20px / semibold |
| 卡片标题 | 16px / medium |
| 正文 | 14px / regular |
| 辅助 | 12px / regular，secondary 色 |
| 代码/SQL | 等宽（如 JetBrains Mono / ui-monospace）14px，浅灰底代码块 |

中文优先系统字体栈：`"PingFang SC", "Microsoft YaHei", sans-serif`。

## 5. 组件观感约定

### 5.1 按钮

| 类型 | 使用 |
|------|------|
| Primary | 页内唯一主行动：新建、保存、立即执行 |
| Default | 次要：取消、测试连接 |
| Dashed/Link | 三级 |
| Danger | 删除、吊销 Token |

**立即执行 / 重跑**：Primary，但点开后 **Modal 确认**（文案含「将发往真实通道」）。

### 5.2 表格

- 表头固定浅底、字重 medium
- 行高舒适（约 48–56px），支持横向滚动
- 操作列右固定（编辑 / 更多）
- 空态：居中短文案 + 主按钮（如「新建数据源」）

### 5.3 表单

- 标签在上（垂直表单），复杂页可两列
- 必填 `*` 
- 错误：字段下红字，不要只靠 Toast
- 密码：圆点 + 脱敏展示已保存状态

### 5.4 Tabs（任务编辑 U2）

- 顶栏 Tabs：基本 | 取数 | 渲染 | 通道 | 调度 | 运行
- 未保存切换：若脏数据则提示（一期可简单「是否离开」）
- 当前 Tab 下主按钮仍以「保存」为主；「立即执行」放在 **运行** Tab + 页头也可放快捷入口

### 5.5 代码/SQL 区（U3）

```text
┌─ 取数 ─────────────────────────────────────┐
│  数据源  [下拉________________v]            │
│  SQL                                        │
│  ┌────────────────────────────────────────┐ │
│  │ SELECT ...          （等宽、浅灰底）     │ │
│  │                                        │ │
│  └────────────────────────────────────────┘ │
│  说明：可用参数 {{biz_date}} …              │
└─────────────────────────────────────────────┘
```

最小高度约 200px，可拖拽增高更佳（二期）。

### 5.6 渲染配置（四形态）

用 **可选卡片组**，不要四个无关表单堆叠：

```text
□ 文本/Markdown    □ 图片    □ 卡片    □ 文件
┌─ 已选：文本 ─────┐  ┌─ 已选：图片 ─────┐
│ 模板 textarea    │  │ 图表/表格样式…   │
└──────────────────┘  └──────────────────┘
```

选中才展开配置，降低噪音。

### 5.7 执行详情时间线

垂直步骤：

```text
● 取数     成功  1.2s   返回 120 行
● 渲染     成功  0.8s   文本+图片+附件
● 投递钉钉 失败  —     错误摘要 [展开]
```

失败步骤红色节点；skipped 灰色「已跳过（无数据）」。

## 6. 关键屏「效果说明」（线框级）

### 6.1 推送任务列表

- 顶：标题「推送任务」+ 右上 Primary「新建任务」
- 下：筛选条（状态、关键字）+ 表格
- 行内：启用 Switch；操作「编辑 | 执行」
- 「执行」→ 确认 Modal → 跳转或 Toast + 链到执行记录

### 6.2 任务编辑（Tabs）

- 顶：任务名（可编辑或只读标题）+ 启用 Switch + 保存
- Tabs 内容区白卡片
- **运行** Tab：大按钮「立即执行」+ 最近 5 次 Run 迷你表（状态徽章）

### 6.3 执行记录列表

- 筛选条突出 **状态** 多选
- 状态列用徽章，失败行可轻微浅红底（可选，注意勿过刺激）
- 操作：「详情 | 重跑」

### 6.4 执行详情

- 顶：状态大徽章 + 重跑按钮
- 左/上：时间线
- 下：Tabs 或折叠：预览 | 日志 | 配置快照
- 图片预览：缩略图点击放大（Modal）
- 文件：下载链接按钮

### 6.5 数据源 / 通道

- 标准「列表 + 抽屉或独立表单页」
- 建议：**独立表单页**（字段多、测连/试发流程更清晰），不用过窄抽屉

---

## 7. 反馈与状态

| 场景 | 表现 |
|------|------|
| 列表加载 | 表格 Skeleton 或居中 Spinner |
| 保存成功 | 顶 Toast 成功 |
| 测连/试发中 | 按钮 loading，防重复点 |
| 测连失败 | 表单项下或 Alert 展示原因 |
| 执行中 | 列表 running 徽章；详情自动刷新（轮询 2–3s，一期可接受） |
| 全局错误 | 页级 Alert 或统一错误页 |

---

## 8. 与实现组件库的关系

| 建议 | 说明 |
|------|------|
| 若 React 管理台 | Ant Design 或 shadcn+Tailwind 均可；**按本视觉 token 定制** |
| 若纯 Python 模板 | 可用 Naive UI / Element 类库同等映射 |
| 不在此文档锁定组件库 | 架构已定 Python；前端栈实现计划再定 |

视觉验收标准：**像同一产品**（间距、徽章、按钮层级一致），而不是某库默认皮肤裸奔。

---

## 9. 明确避免（反模式）

- 大面积紫粉渐变、玻璃拟态营销风
- 每页多种主色按钮抢焦点
- 表格字号过小（&lt;12）硬塞数据
- 失败只 Toast 一闪而过、详情无日志
- 密钥明文回显

## 10. 视觉开放项结论（2026-07-16，V1–V3 均按默认）

| ID | 最终结论 |
|----|----------|
| **V1** | 主色采用 **通用蓝**；若有公司 VI 可在上线前替换 token |
| **V2** | 任务/数据源/通道表单均为 **独立页**（不用主流程大抽屉） |
| **V3** | **一期不做** 深色模式 |
