# 企业数据推送中台

> 项目记忆入口。设计期结论、需求调研、架构决策统一沉淀于此。

## 已确认约束

- **类型**：数据中心推送中台（取数 → 渲染 → 通道）
- **场景**：数据播报 + 配置化 SQL/命令类问题推送（同一流水线）
- **核心原则**：**消息与接收方独立**；接收方 = 通道配置
- **数据源**：**多数据库插件化**；一期 **Doris + MySQL**；统一接口后续扩展
- **消息渲染（均必须）**：文本/Markdown、图片、结构化卡片、文件附件
- **一期通道**：**仅钉钉**；通道 **插件化** 可扩企微等
- **触发/调度**：**混合 C** = 自建调度(B) + DS 经 HTTP 触发(A1)；执行始终在本系统
- **权限**：一期全管理员；分级权限后期再定（设计预留扩展）
- **一期成功标准**：A–F 全要（数据源、渲染、通道、自建调度、DS HTTP、日志重跑）
- **技术栈**：Python 为主
- **元数据库**：MySQL 8（系统配置/执行日志；非业务 Doris）
- **文档记忆**：本库 `20_项目/企业数据推送中台/`
- **调研状态**：P0 已收敛；领域/架构 **已确认**
- **架构要点**：统一 Execution；三插件 SPI；元库 MySQL；混合调度 C
- **代码仓库**：https://github.com/nh1571/data-push-platform（Public；本地可 `/Users/hello/grok/data-push-platform`）
- **默认分支**：`main`；协同规范见仓库 `docs/COLLAB_GIT.md` 与技能 `data-push-collab`
- **本笔记在仓库中的副本**：`docs/product/`（给其他 Agent / 协作者用，不必装 Obsidian）
